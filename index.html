如何获取 APIKey

进入后台后，在左侧菜单打开 API 密钥，然后点击 创建密钥。

如何获取配置

创建密钥后，点击 使用密钥，即可获取对应的配置代码。

配置 Codex CLI / Claude Code CLI

建议先安装并验证 CLI，再分别配置 Codex CLI 与 Claude Code CLI。两者使用的配置方式不同，不建议混在同一个配置文件里。

准备 Node.js 环境

确认本机已安装 Node.js 与 npm。Claude Code 官方要求 Node.js 18 或更高版本。

node -v
npm -v

安装 CLI

按需安装 Codex CLI 和 Claude Code CLI。

# 安装 Codex CLI
npm i -g @openai/codex

# 安装 Claude Code CLI
npm install -g @anthropic-ai/claude-code

验证安装结果

确认命令可以正常被系统识别。

codex --version
claude --version

配置 Codex CLI

创建配置目录，打开或创建 Codex CLI 配置文件 ~/.codex/config.toml，写入下面配置。

mkdir -p ~/.codex
nano ~/.codex/config.toml
# ~/.codex/config.toml
model_provider = "ojbk"
model = "gpt-5.4"
review_model = "gpt-5.4"
model_reasoning_effort = "xhigh"
disable_response_storage = true
network_access = "enabled"
windows_wsl_setup_acknowledged = true
model_context_window = 1000000
model_auto_compact_token_limit = 900000

[model_providers.ojbk]
name = "OJBK API"
base_url = "https://ojbkapi.com"
env_key = "OPENAI_API_KEY"
wire_api = "responses"
requires_openai_auth = true

如果要使用 Japan 或 Singapore 端点，把 base_url 替换成对应地址即可。

配置 Codex CLI 密钥

把后台创建的 API Key 配置为环境变量。临时测试可以直接在当前终端执行；长期使用建议写入你的 Shell 配置文件。

export OPENAI_API_KEY="sk-*********"

配置 Claude Code CLI

Claude Code CLI 不读取 Codex 的 TOML 配置。目标端点需要支持 Claude Code 使用的 Anthropic Messages API，或由网关完成协议转换。

export ANTHROPIC_BASE_URL="https://ojbkapi.com"
export ANTHROPIC_AUTH_TOKEN="sk-*********"
export ANTHROPIC_MODEL="gpt-5.4"

启动验证

任选其一运行，确认 CLI 能正常启动并访问配置好的端点。

# 启动 Codex
codex

# 启动 Claude Code
claude

如何配置 VSCode

VSCode 里建议使用 OpenAI 官方 Codex 扩展。扩展会读取 Codex CLI 的共享配置，因此先完成上一节的 Codex CLI 配置，再安装 VSCode 扩展。

确认 Codex CLI 配置可用

先在终端确认 Codex CLI 能正常启动，并且已经配置好 ~/.codex/config.toml 与 OPENAI_API_KEY。

codex --version
codex

安装官方 VSCode 扩展

打开 VSCode 扩展市场，搜索 Codex - OpenAI's coding agent，确认发布者为 OpenAI 后安装。

扩展市场搜索：
Codex - OpenAI's coding agent

打开 Codex 面板

安装完成后，VSCode 侧边栏会出现 Codex 图标。点击图标打开 Codex 面板，然后按提示启动会话。

检查共享配置

如果 VSCode 里没有使用你的自定义端点，回到终端检查 Codex 配置文件，重点确认 model_provider、base_url 和 env_key。

# ~/.codex/config.toml
model_provider = "ojbk"

[model_providers.ojbk]
base_url = "https://ojbkapi.com"
env_key = "OPENAI_API_KEY"

常见问题

如果扩展无法连接，优先检查 VSCode 是否继承了终端环境变量。必要时重启 VSCode，或在系统 Shell 配置文件里长期保存 OPENAI_API_KEY。

# zsh 示例
echo 'export OPENAI_API_KEY="sk-*********"' >> ~/.zshrc
source ~/.zshrc

如何配置 OpenClaw【小龙虾】

输入命令

在已经安装的小龙虾中，直接输入命令修改配置。

# 修改 OpenClaw 配置
openclaw configure

选择配置项

选择 Model (Pick provider + credentials)。

选择自定义服务商

选择 Custom Provider (Any OpenAI or Anthropic compatible endpoint)。

配置 API Base URL

https://ojbkapi.com/v1

粘贴 API Key

选择 Paste API key now (Stores the key directly in OpenClaw config)，然后粘贴后台创建的 API Key。

sk-*********

选择 API 端点

选择 OpenAI-compatible (Uses /chat/completions)。

选择模型

粘贴模型名称。

gpt-5.4

完成配置

后续一直回车即可。出现 Configured custom provider: *************/gpt-5.4，说明已经配置成功。再次出现 Select sections to configure 时，选择 Continue 结束配置。
