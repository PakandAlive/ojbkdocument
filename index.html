产品说明文档

指南

快速开始

这是一份 GitBook 风格的静态说明文档示例。你可以直接修改 HTML 内容，也可以后续迁移到 VitePress、Docusaurus 或 MkDocs Material。

当前版本不需要服务器，直接打开 index.html 就能预览，也可以部署到 GitHub Pages。

安装与部署

这个测试版没有依赖，不需要执行安装命令。目录结构如下：

gitbook-docs-demo/
├── index.html
├── README.md
└── assets/
    ├── app.js
    └── styles.css

基础配置

你可以优先修改这些内容：

  • 文档标题：修改 index.html 里的 产品说明文档
  • 左侧目录：修改 <nav class="toc"> 下的链接
  • 正文内容：按 <section> 追加章节
  • 主题样式：修改 assets/styles.css

使用流程

  1. 先把每个章节的大纲写清楚。
  2. 再补充截图、配置示例和常见错误说明。
  3. 最后提交到 GitHub 仓库并开启 GitHub Pages。

常见问题

是否需要服务器？

不需要。GitHub Pages 会托管静态网站。

是否可以绑定自己的域名？

可以。GitHub Pages 支持自定义域名，后续在仓库设置里配置即可。

是否支持搜索？

当前示例提供前端页面内搜索。正式文档站可以接入更完整的本地搜索方案。

发布到 GitHub Pages

上传到 GitHub 后，在仓库设置中开启 Pages：

  1. 进入仓库的 Settings。
  2. 打开 Pages。
  3. Source 选择 Deploy from a branch。
  4. Branch 选择 main，目录选择 /root。
  5. 保存后等待 GitHub 发布。

发布完成后的访问地址通常是：

https://你的用户名.github.io/仓库名/
